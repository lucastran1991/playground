### Overview
- **Priority:** P2
- **Status:** completed
- **Description:** Add Swagger API documentation to Go backend using swaggo/swag. Annotate auth handlers, generate docs, serve Swagger UI at `/swagger/*`.

### Key Insights
- `swaggo/swag` generates OpenAPI 2.0 spec from Go comments
- `gin-swagger` serves the Swagger UI as middleware
- Only 5 endpoints to annotate: health, register, login, refresh, me
- Swagger docs auto-generated into `backend/docs/` (swagger.json + swagger.yaml + docs.go)

### Requirements
**Functional:**
- Swagger UI accessible at `http://localhost:8080/swagger/index.html`
- All 5 endpoints documented with request/response schemas
- "Try it out" works for public endpoints

**Non-functional:**
- `swag init` generates without errors
- Generated docs committed to repo (so users don't need swag CLI to run)

### Architecture
```
backend/
  docs/                    # NEW -- generated by swag
    docs.go
    swagger.json
    swagger.yaml
  cmd/server/main.go       # Add swag general API info comments
  internal/handler/        # Add endpoint annotations
  internal/router/         # Add swagger UI route
```

### Dependencies to ADD
```bash
# CLI tool (dev only, install globally or via go install)

# Go packages
go get github.com/swaggo/gin-swagger
go get github.com/swaggo/files
```

### Files to MODIFY

**1. `backend/cmd/server/main.go`**
Add general API info comments above `main()`:
```go
// @title       MyApp API
// @version     1.0
// @description Starter template API with JWT authentication
// @host        localhost:8080
// @BasePath    /
// @securityDefinitions.apikey BearerAuth
// @in header
// @name Authorization
```

Import the generated docs package:
```go
import _ "myapp/docs"
```

**2. `backend/internal/handler/auth_handler.go`**
Add swag annotations above each handler method:

- `Register`: `@Summary Register`, `@Tags auth`, `@Accept json`, `@Produce json`, `@Param body body RegisterRequest true "Registration data"`, `@Success 201`, `@Failure 400`, `@Router /api/auth/register [post]`
- `Login`: similar pattern with `LoginRequest` + `AuthResponse`
- `RefreshToken`: `@Param body body RefreshRequest true "Refresh token"`, `@Router /api/auth/refresh [post]`
- `Me`: `@Security BearerAuth`, `@Router /api/auth/me [get]`

Note: Check actual request/response struct names in `auth_handler.go` -- use the real types for `@Param body` and `@Success`.

**3. `backend/internal/router/router.go`**
Add swagger UI route:
```go
import (
    swaggerFiles "github.com/swaggo/files"
    ginSwagger "github.com/swaggo/gin-swagger"
)

// Inside Setup(), after health check:
r.GET("/swagger/*any", ginSwagger.WrapHandler(swaggerFiles.Handler))
```

**4. `backend/go.mod`**
After adding deps, run `go mod tidy`. New deps expected:
- `github.com/swaggo/swag`
- `github.com/swaggo/gin-swagger`
- `github.com/swaggo/files`

### Implementation Steps

1. Install swag CLI: `go install github.com/swaggo/swag/cmd/swag@latest`
2. Add Go package deps: `cd backend && go get github.com/swaggo/gin-swagger github.com/swaggo/files`
3. Add general API info comments to `main.go`
4. Add `import _ "myapp/docs"` to `main.go`
5. Annotate each handler in `auth_handler.go` with swag comments
6. Add swagger UI route to `router.go`
7. Generate docs: `cd backend && swag init -g cmd/server/main.go -o docs/`
8. Run `go mod tidy`
9. Verify: `go build ./cmd/server && ./server` then open `http://localhost:8080/swagger/index.html`

### Risk Assessment
- **swag CLI version**: Pin to latest stable. Breaking changes rare but possible.
- **Generated docs size**: Adds ~3 files to `backend/docs/`. Acceptable.
- **Request/response types**: Must match actual Go structs. If handlers use inline types, may need to extract named structs for swag to reference.

### Security Considerations
- Swagger UI exposes API surface. For production, consider disabling behind env flag.
- Document in README: "Swagger UI is enabled by default; disable in production by removing the route."
